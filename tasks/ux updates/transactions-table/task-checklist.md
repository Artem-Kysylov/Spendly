Чеклист: Таблица транзакций (адаптация под мобильные)

1. Разделить рендеринг по брейкпоинтам:
   - Desktop: рендер текущего DataTable (`hidden md:block`).
   - Mobile: рендер нового списка карточек (`block md:hidden`).

2. Создать компонент `MobileTransactionCard`:
   - Верстка: flex (row).
   - Слева: аватар/иконка категории (круглый), emoji/цвет бюджета.
   - Центр: сверху — название (font-medium, truncate), снизу — дата + категория (text-muted-foreground, text-xs).
   - Справа: сумма (font-bold), `text-error` для расхода, `text-success` для дохода.

3. Создать компонент `MobileTransactionsList`:
   - Пропсы: `transactions`, `onDeleteTransaction(id)`, `onEditTransaction(payload)`.
   - Сортировка по `created_at` (по убыванию), рендер карточек, пустые состояния и лоадер.

4. Интегрировать условный рендеринг на странице транзакций:
   - Desktop: текущий DataTable.
   - Mobile: `MobileTransactionsList`.

5. Действия карточки:
   - Кнопка «три точки» с пунктами `Edit` и `Delete`.
   - Подтверждение удаления (модалка), обновление списка после операций.

6. Свайп‑действия (опционально):
   - Добавить свайп влево для удаления с использованием `motion/react`.
   - Настроить порог, анимацию и отмену свайпа.

7. Локализация:
   - Использовать существующие ключи из `transactions.*` и `common.*` для кнопок и состояний.
   - Новые строки не добавлять.

8. Стили и доступность:
   - Единые цвета для типа (`text-error`/`text-success`), контраст и читаемость.
   - `aria-label` для кнопок действий, фокус‑стили, `truncate` для длинных названий.

9. Тестирование:
   - Проверка на разных ширинах экрана (xs/sm/md).
   - Длинные названия, отсутствие категории/эмодзи, большие списки (200+ записей).
   - Производительность и отсутствие «дёрганий» при скролле.
